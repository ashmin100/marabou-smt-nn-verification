Marabou를 활용한 신경망 검증: MNIST 분류기의 국소적 강건성(Local Robustness)
===================================================================================

1. 모델, 데이터셋 및 검증 쿼리(Verification Query)
------------------------------------------

모델 (Model):
  구조(Architecture): 완전연결 신경망 (MLP)
                  입력 레이어: 784 뉴런 (28×28 MNIST 픽셀 크기)
                  은닉 레이어 1: 64 뉴런, ReLU 활성화 함수
                  은닉 레이어 2: 32 뉴런, ReLU 활성화 함수
                  출력 레이어: 10 뉴런 (숫자 클래스 0–9)
  총 파라미터 수: 약 55,000개
  포맷: ONNX (opset 11), torch.onnx.export를 통해 추출됨
  학습 정보: MNIST 학습 세트(60,000개 이미지)에서 5 에폭 동안 Adam 옵티마이저와 교차 엔트로피 손실 함수로 학습됨. 테스트 정확도: 약 97.5%

  해당 모델 선정 이유:
  이 네트워크는 Marabou의 완전(complete) SMT 기반 검증기가 합리적인 시간(쿼리당 약 5분 이내) 내에 연산을 마칠 수 있을 만큼 작습니다. 더 큰 모델(예: 전체 ResNet)의 경우 솔버가 타임아웃을 발생시킬 수 있습니다. ONNX 형식은 Marabou 2.0에서 기본으로 지원하는 포맷이며, 파이썬 API(maraboupy)를 통해 직관적으로 사용할 수 있습니다. MNIST는 국소적 강건성 검증의 표준 벤치마크이며, 실제 라벨이 명확하기 때문에 선택되었습니다.

데이터셋 (Dataset):
  MNIST 테스트 세트 (10,000개 이미지). 픽셀 값은 평균=0.1307, 표준편차=0.3081로 정규화되어 약 [-2.8, 2.8] 범위 내에 위치합니다.

검증 쿼리 (Verification Query):
  숫자 d로 올바르게 분류된 테스트 이미지 x가 주어졌을 때, 다음을 검증합니다:

      ||x' - x||_inf <= epsilon 을 만족하는 모든 x'에 대하여,
      네트워크가 x' 역시 숫자 d로 분류하는가?

  이는 SAT(충족 가능성) 문제로 공식화됩니다: Marabou는 epsilon-ball 반경 내에서 네트워크가 예측값 j ≠ d를 출력하게 만드는 입력 x'를 탐색합니다.
  - UNSAT: 조건을 만족하는 x'가 존재하지 않음 → 네트워크가 수학적으로 강건(robust)함이 증명됨.
  - SAT: 적대적 예제(counterexample)가 발견됨 → 강건성 속성이 위반됨.

  입력 제약조건: 모든 i에 대해 x'[i] ∈ [x[i] - epsilon, x[i] + epsilon]
  출력 제약조건: output[j] >= output[d] 를 만족하는 j ≠ d 가 존재함 (논리합(disjunction)으로 인코딩됨)


2. 검증 결과 (Results)
-----------

실험 1 — 작은 epsilon (강건한 경우)
  샘플       : 테스트 이미지 인덱스 0, 실제 라벨 = 7
  Epsilon    : 0.01
  결과       : UNSAT
  소요 시간  : 0.03 초
  해석 (Interpretation):
    네트워크는 해당 입력 주변 반경 0.01의 L-inf 범위 내에서 강건함이 확실하게 증명되었습니다. 크기 0.01 이하의 어떠한 적대적 섭동(perturbation)도 예측된 클래스를 바꿀 수 없습니다.

실험 2 — 중간 epsilon
  샘플       : 테스트 이미지 인덱스 0, 실제 라벨 = 7
  Epsilon    : 0.05
  결과       : UNSAT
  소요 시간  : 0.08 초
  해석 (Interpretation):
    섭동 허용 범위를 5배 늘렸음에도(0.05), Marabou는 전체 공간을 완전히 탐색하여 적대적 예제가 없음을 공식적으로 입증했습니다. 이 모델은 해당 샘플에 대해 매우 높은 수준의 국소적 강건성을 보여줍니다.

실험 3 — 경계 탐색 (큰 epsilon)
  샘플       : 테스트 이미지 인덱스 0, 실제 라벨 = 7
  Epsilon    : 0.12 ~ 0.15 (UNSAT) / 0.20 (TIMEOUT)
  소요 시간  : 약 13초 (0.15 기준) / 300.0초 (0.20 기준)
  해석 (Interpretation):
    정규화된 물리적 픽셀 경계([-0.42, 2.82])로 섭동 범위를 클리핑(Clamping)한 결과, epsilon=0.15까지 단 13초 만에 완벽한 강건성(UNSAT)이 증명되었습니다. 클리핑 전에는 솔버가 현실에 존재할 수 없는 유령 픽셀(255 초과 값)을 탐색하느라 0.12 검증에 248초를 낭비했고, 0.15에서는 거짓 취약점(False SAT)을 보고했습니다. 물리적 한계를 적용하여 탐색 공간을 최적화한 성과가 명확히 드러나며, epsilon 0.20에서는 연산 폭발로 인해 타임아웃이 발생했습니다.


3. 논의: Marabou의 강점과 한계
----------------------------------------------------

강점 (Strengths):
  (a) 공식적 완전성 (Formal completeness). FGSM이나 PGD와 같은 기울기 기반 공격 기법과 달리, Marabou는 수학적 보장을 제공합니다. UNSAT 결과는 단지 테스트한 샘플만이 아니라, 해당 입력 공간 전체에 걸쳐 속성이 성립함을 의미합니다. 이는 안전 필수(safety-critical) 애플리케이션에 매우 중요합니다.
  (b) 적대적 예제(Counterexample) 생성. 속성이 위반될 때(SAT), Marabou는 실제 적대적 입력값을 반환하므로, 어떤 부분에서 실패가 발생했는지 해석하고 조치할 수 있습니다.
  (c) 표준 포맷 지원. Marabou 2.0은 ONNX를 기본 포맷으로 지원하여, torch.onnx.export 등을 통한 PyTorch 및 TensorFlow 워크플로우와 자연스럽게 통합됩니다.
  (d) VNNLIB 지원. Marabou는 VNN-COMP 속성 형식을 지원하여 표준 검증 대회의 벤치마크에도 쉽게 참여할 수 있습니다.

한계 (Limitations):
  (a) 확장성 (Scalability). 이번 실험에서 겪은 가장 큰 한계는 연산 비용이었습니다. (784→64→32→10) 크기의 작은 네트워크에서도 epsilon=0.01일 땐 0.03초가 걸렸지만, epsilon을 0.20으로 늘리자 300초 타임아웃에 도달했습니다. 전체 크기의 네트워크(ResNets, VGGs)는 현재의 완전 검증 방식으로는 처리할 수 없습니다. 뉴런의 개수는 Marabou가 풀어야 할 LP/SMT 문제의 크기와 직결됩니다.
  (b) 설치의 복잡성. macOS 환경에서 소스 코드로 Marabou를 빌드하려면 Cmake, Protobuf, pybind11 경로 문제, Homebrew를 통한 Boost 설치 등 여러 의존성 문제를 해결해야 했습니다. 이러한 요소들은 단순한 pip 설치 지원 도구보다 도입 장벽을 높입니다.
  (c) 지원하는 활성화 함수. Marabou의 완전 모드(complete mode)는 조각적 선형(piecewise-linear) 활성화 함수(예: ReLU)에서 가장 잘 작동합니다. Sigmoid, tanh, GELU 등은 완전성을 희생하는 추상화 기반 방법론을 필요로 합니다.
  (d) 빡빡한 epsilon 예산. 정규화된 입력값을 갖는 MNIST의 경우, epsilon=0.01은 실제 원시 픽셀 공간에서 약 (2.5/255)의 매우 작은 섭동에 불과함에도 검증에 상당한 시간이 소요됩니다. 하드웨어가 제한적일 경우 더 큰 epsilon 값은 필연적으로 타임아웃을 초래합니다.

개인적 관찰 (Personal Observations):
  이번 실험은 공식 검증(Formal Verification)에서 도메인 지식(물리적 픽셀 경계)의 적용이 얼마나 치명적으로 중요한지 명확히 보여주었습니다. 입력값에 물리적 한계를 걸지 않았을 때, SMT 솔버는 현실에 존재 불가능한 가짜 입력값을 탐색하며 엄청난 연산량을 낭비했습니다. (0.12 검증에 248초 소요 및 0.15에서 False SAT 도출). 그러나 단순한 픽셀 클리핑 제약 조건을 추가하자 0.12 검증 시간이 6초로 40배나 단축되었고, 0.15에서도 모델이 완벽히 강건함(UNSAT)을 빠르고 정확하게 증명해 냈습니다. 이는 솔버의 알고리즘적 스펙 못지않게 "탐색 공간의 현실적 최적화"가 필수적임을 시사합니다.
  
  또한, 소프트웨어 엔지니어링 관점에서 Marabou 2.0으로의 API 전환(결과값을 튜플에서 리스트로 반환하고 Equation 상수 경로가 변경된 점 등)은 안전 필수 시스템 검증 코드를 작성할 때 개발자가 API 파싱 및 업데이트에 얼마나 극도로 주의를 기울여야 하는지 보여주는 좋은 사례였습니다.
